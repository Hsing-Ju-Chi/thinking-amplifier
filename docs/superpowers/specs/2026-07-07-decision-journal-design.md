## 一、目的

v1 誠實承認的邊界：快判讓使用者「當下判斷得更好」，但真正的判斷力成長需要**跟現實對答案**。v2 補上這個迴圈：

- **判斷日誌**：記錄使用者的裁決與信心 → 事後與實際結果對照 → 給出校準回饋（找系統性偏差，不是算對錯）
- **心法筆記**：把每次快判的判斷心法累積成使用者的個人判斷力字典

兩者共用本機檔案機制。此功能僅在有檔案工具的平台完整運作（Claude Code、Codex CLI 等）；貼上型平台誠實降級。

## 二、實作方式

方案 A：全部內建 SKILL.md（新增一節約 40 行），維持單一檔案、零依賴、跨平台的哲學。不拆參考檔、不附腳本。

## 三、檔案佈局

```
~/.thinking-amplifier/          # 使用者資料，與 skill 安裝目錄分離（安裝目錄是 git repo，更新不得動使用者資料）
├── decision-journal.md         # 判斷日誌
└── heuristics.md               # 心法筆記
```

首次寫入時自動建立目錄與檔案。

## 四、記錄時機（可操作判準，防弱模型濫問）

AI 主動問「要把這筆記進判斷日誌嗎？之後可以復盤對照」的條件——**兩者同時滿足**：

1. 判斷有**未來可驗證的結果**（決策、預測、方案選擇——做下去會知道對錯）
2. 使用者**已說出自己的裁決**（等裁決出現才問，不是分析完就問）

明確不問的情境：純審稿（改文案、看報告等無後續結果可對照者）；同一對話已被拒絕過一次。

主動指令永遠有效：「記下來」→ 直接寫日誌；「記下這條心法」→ 寫 heuristics.md。

## 五、條目格式

decision-journal.md 條目：

```markdown
## 2026-07-07｜Reels 預算配置
- 判斷事項：是否整季 all-in IG Reels
- 我的裁決：改 80/20，四週後看數據再加碼
- 信心：中高
- AI 當時立場：方向對，all-in 論據不足
- 驗證時點：約四週後（8 月初）
- 結果：（待填）
```

「信心」與「驗證時點」為必填——缺了就只是回憶錄，不是校準訓練。

heuristics.md 條目（一行一條）：

```markdown
- 2026-07-07｜數字很精確，不代表來源可靠（源自：手搖飲市場報告快判）
```

## 六、復盤流程（使用者說「復盤」時）

1. 讀 decision-journal.md，列出「驗證時點已到且結果待填」的條目（無條目則如實說，不硬編）
2. 逐筆詢問實際結果，填回檔案
3. **校準回饋**：對照當時信心與實際結果，指出**系統性偏差**（例：「高信心判斷 3 中 2」「時間壓力下的判斷普遍過度自信」），不是逐題計分
4. 回顧 heuristics.md：這段時間哪些心法實際用上了
5. 責任交還收尾：「下一步怎麼調，你決定」

## 七、跨平台降級（誠實邊界）

- 無檔案工具的平台：不提日誌功能；使用者主動說「記下來」時，輸出格式化條目請使用者自行貼到筆記，明說「我在這個環境沒有辦法幫你保存」——不得假裝有記憶
- README 明示：判斷復盤需要有檔案能力的環境
