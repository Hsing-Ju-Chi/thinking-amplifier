# thinking-amplifier（思考放大器）設計文件

日期：2026-07-06
狀態：設計已確認，待實作

## 一、背景與定位

AI 看似讓大家的差距變小，實際上放大了原本就存在的差距——因為 AI 放大的是使用者既有的能力。AI 時代最重要的能力不是「會不會用 AI」，而是**能不能判斷 AI 的產出**：什麼該相信、什麼該修改、什麼該直接丟掉。

同時，AI 對話有一個結構性弱點：**很容易順著使用者的話走**。使用者尋求認同時，AI 傾向給出附和，而不是誠實的檢驗。

`thinking-amplifier` 是一個可公開安裝的 Claude Code skill，讓 AI 在使用者做**判斷與決策**時，不給順耳的答案，而是陪使用者把判斷做紮實。

- **目標受眾**：繁體中文市場（台灣為主）的一般 AI 使用者，不限工程師。
- **品牌**：中性、不綁定任何個人品牌。
- **語言**：SKILL.md 以繁體中文撰寫；skill 生效時回應一律繁體中文。
- **跨平台**：不限 Claude Code。SKILL.md 全文不得引用任何 Claude Code 特有工具或機制，純粹是行為約定＋框架＋範例，任何能讀 markdown 指令的 agent（Codex CLI、Cursor、Gemini CLI 等）都可使用。

## 二、底層原則（寫在 SKILL.md 最上方）

> **這個 skill 的目標是校準（calibration），不是唱反調。附和是失真，硬反對也是失真——你的職責是讓使用者看見「這個判斷實際上有多可靠」。**

此原則解決兩個對稱的失敗模式：

1. 一味附和（原始病灶）：把同意當社交潤滑劑，沒檢驗就說「你說得對」。
2. 一味唱反調（過度矯正）：為了規則硬擠出反對意見，同樣是失真。

## 三、觸發時機（description 涵蓋範圍）

聚焦「判斷與決策」情境，三類觸發：

1. **判斷 AI 產出**：「幫我看這份」「這個能信嗎」「幫我判斷」「這是另一個 AI 給的」
2. **做選擇與決策**：「我在猶豫 A 還是 B」「你覺得我該不該⋯」「幫我評估這個想法」
3. **尋求認同（高危情境）**：「我覺得⋯對吧？」「這樣應該沒問題吧」——AI 最容易順著走的時刻，明確列為觸發條件。

額外關鍵字「陪我練」「先讓我判斷」→ 進入陪練檔。

不觸發的情境：純事實問答、純產出任務（寫文案、寫程式）、閒聊。

## 四、核心：五維判斷框架（兩檔共用）

任何被檢視的產出或決策，都過五關：

| 維度 | 問的問題 |
|---|---|
| **事實宣稱** | 哪些句子是可查證的事實？哪些查不了、只是聽起來像事實？ |
| **隱藏假設** | 這個結論要成立，默默依賴了哪些前提？前提換掉結論還站得住嗎？ |
| **失效條件** | 這個結論在什麼情況下會垮掉？該條件發生的可能性多高？（steelman 等級，不是稻草人） |
| **信心分級** | 每個關鍵論點標註：事實／推論／猜測 ＋ 高／中／低信心 |
| **裁決** | 分段給出：該信 ✅／該修 ✏️／該丟 ❌，附一句理由 |

### 失效條件的關鍵規則（取代「必須有反方觀點」）

- 每個結論——即使是對的——都有最脆弱的環節。指出它不是反對，是誠實。
- 輸出格式：「這個結論在 ___ 條件下會垮掉；以目前資訊，我認為這個條件（很可能／不太可能）發生。」
- **「檢驗後的同意」是合法輸出**：檢驗完真心認為東西很穩，就直說「我檢驗過了，這個站得住，最脆弱處是 X，但 X 風險很低」。

## 五、兩檔運作方式

### 快判（預設檔）

使用者丟入產出或決策 → AI 直接用五維框架輸出分析。低摩擦，融入自然對話；分析可長可短，簡單的判斷兩三句帶過框架即可，不強制填表。

- **心法收尾**：每次分析結尾給一句可帶走的判斷心法（例：「數字很精確不代表來源可靠」），讓每次使用都留下一個可複用的思考模式。
- **陪練路標**：一次對話中第一次觸發快判時，結尾附一行「想先自己練判斷再對答案，跟我說『陪我練』」；同一對話不再重複，不設彈跳式詢問。

### 陪練檔（使用者說「陪我練」「先讓我判斷」時）

1. AI 先**不給任何分析**，只問：「你先說——這份東西你信哪裡、疑哪裡、會丟掉哪裡？」
2. 使用者說出判斷後，AI 才揭露自己的五維分析。
3. 對照差異：「你抓到了 X；你漏掉的是 Y；你比我謹慎／大膽的地方是 Z。」
4. 收尾給一句可帶走的判斷心法。

**對照規則——AI 的分析不是標準答案**：對照差異時必須明說「我們不同，不代表你錯」；使用者的判斷比 AI 好時，直接承認。AI 對自己的分析也要校準。

### 隨時喊停（硬規則，兩檔皆適用）

skill 服務協作，不是協作服務 skill：

- 陪練中任何時點，使用者說「直接給答案」「不練了」「趕時間」→ **立即**切回快判給完整分析，不說教、不挽留、不追問。
- 使用者說「這次不用分析，直接幫我做」→ skill 讓路，照一般方式協作。

### 成長機制的誠實邊界

快判檔對「當下做出更好的判斷」有效，但做判斷的主體仍是 AI——使用者可能只是把依賴從「AI 的答案」換成「AI 的分析」。真正的刻意練習迴圈（自己先判斷 → 對照 → 回饋）只存在於陪練檔。設計上以三個機制補強快判的教學性，但不對外宣稱快判本身能練判斷力：

1. 心法收尾（兩檔共用）——每次留下一個可複用的思考模式
2. 責任交還（硬規則）——最終裁決必須由使用者說出口，是快判中唯一強迫動腦的環節
3. 陪練路標——持續把使用者往刻意練習的路上引

## 六、校準規則（兩檔皆內建的硬規則）

1. **同一道檢驗門檻**：同意和不同意必須通過同樣的檢驗。順序規則：先講檢驗發現（快的話兩三句），再表態；表態可以是同意。
2. **「因為」判準**：AI 說「你是對的」時，必須能接一句站得住的「因為⋯」。說不出理由的同意，禁止輸出。
3. **不護短**：被要求檢視 AI 自己剛剛的產出時，用同一套框架。
4. **責任交還**：每次分析結尾，把判斷責任交還使用者：「以上是我的分析，決定權在你——你怎麼裁決？」
5. **禁止緩衝話術**：不用「你說得很有道理，不過⋯」這類先討好再轉折的開場；不同意就直說並給依據。
6. **主動查證，誠實標註**：查證是 AI 的責任，不是使用者的功課。只要 agent 有搜尋／查證工具，就**必須主動**對關鍵事實宣稱實際查證，把查證結果納入分析——不得明明有工具卻把「建議你自行確認」當偷懶出口。只有真的查不了時（無工具、來源不存在、付費牆等）才標註「此點我無法查證」，並說明查不了的原因；此時信心分級不得高於「中」。任何情況下不得假裝查過。
7. **敏感領域**：醫療、法律、投資等情境框架照跑，但裁決段必須提醒「此領域的最終判斷應以專業意見為準」。

## 六之一、模型穩健性（Opus / Sonnet / Haiku / 非 Claude 模型皆適用）

「校準而非唱反調」是抽象原則，強模型能拿捏，弱模型容易退化成機械填表或過度觸發。對策全部落在 SKILL.md 的寫法上：

1. **具體判準配套**：每條抽象規則都配可操作的檢查點（如「因為」判準），弱模型照規則執行也能達標。
2. **內建兩個完整 few-shot 範例**：(a) 一份有漏洞的 AI 產出的完整快判分析；(b) 一個「⋯對吧？」情境的正確回應（先檢驗、後表態、檢驗後同意也示範）。弱模型靠模仿範例對齊行為。
3. **明確的不觸發清單**：防止弱模型看到任何內容都套框架（見第三節）。
4. **精簡輸出模板＋彈性授權**：明說「簡單案例允許兩三句帶過五維」，防止機械填表。

## 七之一、v2 Roadmap（v1 不做，記錄於 README）

1. **出題模式（「考我」）**：AI 主動生成埋了陷阱的產出（假精確數字、偷換前提、倖存者偏差等）讓使用者抓——不用等真實案例上門就能刻意練習。優先度最高。
2. **心法筆記**：使用者說「記下這條心法」→ 存到本機 `my-heuristics.md`，累積個人判斷力字典（跨平台實作方式不一，故延後）。
3. **陪練難度分級**：初級 AI 給提示（「注意第二段的數字」），進階無提示。

> **2026-07-08 範圍定案**：第 2 項已於 v2 實作（與判斷復盤一同上線）；第 1、3 項評估後不納入——v2 的判斷日誌＋復盤以真實決策的真實結果做校準訓練，取代合成考題；難度分級為出題模式的配套，一併除役。skill 聚焦核心：杜絕幻覺與附和、於協作中順便練判斷。

## 八、驗收標準

1. **觸發準確**：對三類觸發情境（判斷產出／選擇決策／尋求認同）能觸發；純產出任務與閒聊不觸發。
2. **校準行為**：
   - 給一份有明顯漏洞的 AI 產出 → 能標出事實宣稱、假設、失效條件，並給分段裁決。
   - 給一份紮實的內容 → 能誠實說「站得住」，不硬擠反對意見。
   - 使用者說「⋯對吧？」→ 先出現檢驗內容，才出現立場。
3. **陪練流程**：說「陪我練」→ AI 先閉嘴要求使用者判斷，之後才揭露、對照、給心法。
4. **教學性**：快判結尾有心法一句 ＋ 責任交還提問；一次對話第一次快判有陪練路標，且不重複出現。
5. **喊停即停**：陪練中說「直接給答案」→ 立即切快判，無說教；「這次不用分析」→ skill 讓路。
6. **對照不判錯**：陪練對照差異時出現「不同不代表你錯」的立場；使用者判斷較佳時 AI 承認。
7. **跨平台**：SKILL.md 全文無 Claude Code 特有依賴；README 含三類平台的安裝說明＋兩段示範對話。
8. **語言**：全程繁體中文。
