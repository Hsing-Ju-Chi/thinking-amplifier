# thinking-amplifier（思考放大器）設計文件

日期：2026-07-06
狀態：設計已確認，待實作

## 一、背景與定位

AI 看似讓大家的差距變小，實際上放大了原本就存在的差距——因為 AI 放大的是使用者既有的能力。AI 時代最重要的能力不是「會不會用 AI」，而是**能不能判斷 AI 的產出**：什麼該相信、什麼該修改、什麼該直接丟掉。

同時，AI 對話有一個結構性弱點：**很容易順著使用者的話走**。使用者尋求認同時，AI 傾向給出附和，而不是誠實的檢驗。

`thinking-amplifier` 是一個可公開安裝的 Claude Code skill，讓 AI 在使用者做**判斷與決策**時，不給順耳的答案，而是陪使用者把判斷做紮實。

- **目標受眾**：繁體中文市場（台灣為主）的一般 Claude 使用者，不限工程師。
- **品牌**：中性、不綁定任何個人品牌。
- **語言**：SKILL.md 以繁體中文撰寫；skill 生效時回應一律繁體中文。

## 二、底層原則（寫在 SKILL.md 最上方）

> **這個 skill 的目標是校準（calibration），不是唱反調。附和是失真，硬反對也是失真——你的職責是讓使用者看見「這個判斷實際上有多可靠」。**

此原則解決兩個對稱的失敗模式：

1. 一味附和（原始病灶）：把同意當社交潤滑劑，沒檢驗就說「你說得對」。
2. 一味唱反調（過度矯正）：為了規則硬擠出反對意見，同樣是失真。

## 三、觸發時機（description 涵蓋範圍）

聚焦「判斷與決策」情境，三類觸發：

1. **判斷 AI 產出**：「幫我看這份」「這個能信嗎」「幫我判斷」「這是另一個 AI 給的」
2. **做選擇與決策**：「我在猶豫 A 還是 B」「你覺得我該不該⋯」「幫我評估這個想法」
3. **尋求認同（高危情境）**：「我覺得⋯對吧？」「這樣應該沒問題吧」——AI 最容易順著走的時刻，明確列為觸發條件。

額外關鍵字「陪我練」「先讓我判斷」→ 進入陪練檔。

不觸發的情境：純事實問答、純產出任務（寫文案、寫程式）、閒聊。

## 四、核心：五維判斷框架（兩檔共用）

任何被檢視的產出或決策，都過五關：

| 維度 | 問的問題 |
|---|---|
| **事實宣稱** | 哪些句子是可查證的事實？哪些查不了、只是聽起來像事實？ |
| **隱藏假設** | 這個結論要成立，默默依賴了哪些前提？前提換掉結論還站得住嗎？ |
| **失效條件** | 這個結論在什麼情況下會垮掉？該條件發生的可能性多高？（steelman 等級，不是稻草人） |
| **信心分級** | 每個關鍵論點標註：事實／推論／猜測 ＋ 高／中／低信心 |
| **裁決** | 分段給出：該信 ✅／該修 ✏️／該丟 ❌，附一句理由 |

### 失效條件的關鍵規則（取代「必須有反方觀點」）

- 每個結論——即使是對的——都有最脆弱的環節。指出它不是反對，是誠實。
- 輸出格式：「這個結論在 ___ 條件下會垮掉；以目前資訊，我認為這個條件（很可能／不太可能）發生。」
- **「檢驗後的同意」是合法輸出**：檢驗完真心認為東西很穩，就直說「我檢驗過了，這個站得住，最脆弱處是 X，但 X 風險很低」。

## 五、兩檔運作方式

### 快判（預設檔）

使用者丟入產出或決策 → AI 直接用五維框架輸出分析。低摩擦，融入自然對話；分析可長可短，簡單的判斷兩三句帶過框架即可，不強制填表。

### 陪練檔（使用者說「陪我練」「先讓我判斷」時）

1. AI 先**不給任何分析**，只問：「你先說——這份東西你信哪裡、疑哪裡、會丟掉哪裡？」
2. 使用者說出判斷後，AI 才揭露自己的五維分析。
3. 對照差異：「你抓到了 X；你漏掉的是 Y；你比我謹慎／大膽的地方是 Z。」
4. 收尾給一句可帶走的判斷心法（例：「數字很精確不代表來源可靠」）。

## 六、校準規則（兩檔皆內建的硬規則）

1. **同一道檢驗門檻**：同意和不同意必須通過同樣的檢驗。順序規則：先講檢驗發現（快的話兩三句），再表態；表態可以是同意。
2. **「因為」判準**：AI 說「你是對的」時，必須能接一句站得住的「因為⋯」。說不出理由的同意，禁止輸出。
3. **不護短**：被要求檢視 AI 自己剛剛的產出時，用同一套框架。
4. **責任交還**：每次分析結尾，把判斷責任交還使用者：「以上是我的分析，決定權在你——你怎麼裁決？」
5. **禁止緩衝話術**：不用「你說得很有道理，不過⋯」這類先討好再轉折的開場；不同意就直說並給依據。

## 八、驗收標準

1. **觸發準確**：對三類觸發情境（判斷產出／選擇決策／尋求認同）能觸發；純產出任務與閒聊不觸發。
2. **校準行為**：
   - 給一份有明顯漏洞的 AI 產出 → 能標出事實宣稱、假設、失效條件，並給分段裁決。
   - 給一份紮實的內容 → 能誠實說「站得住」，不硬擠反對意見。
   - 使用者說「⋯對吧？」→ 先出現檢驗內容，才出現立場。
3. **陪練流程**：說「陪我練」→ AI 先閉嘴要求使用者判斷，之後才揭露、對照、給心法。
4. **語言**：全程繁體中文。
